◉ Configuración en GHL
1

Custom Action en GHL

Ve a Settings → Integrations → Custom Actions y crea una nueva acción de tipo Webhook.

2

URL del webhook

Apunta la acción a tu servidor:
http://TU-SERVIDOR/ghl/call

3

Campos a enviar

Mapea contact.phone → contact_phone y opcionalmente user.phone → agent_number.

4

Asignar al pipeline

En el contacto del pipeline, añade la acción al botón de teléfono o como acción rápida.

◇ Endpoints disponibles
POST /ghl/call
Endpoint principal. GHL lo llama al hacer click en el teléfono del pipeline. Inicia la llamada vía Ringover.
// Body esperado:
{
  "contact_phone": "+34612345678",  // requerido
  "agent_number": "+34911234567",   // opcional
  "contact_name": "Juan García",       // opcional
  "contact_id": "ghl_contact_id"      // opcional
}
POST /ghl/webhook
Webhook genérico. Acepta directamente el payload nativo de GHL sin transformación.
GET /ringover/users  ·  /ringover/numbers  ·  /ringover/calls
Proxies de consulta a la API de Ringover. Útiles para poblar selectores en GHL.
◈ Ejemplo de integración en GHL (Custom Action)
# GHL Custom Action → HTTP Request
# Method: POST
# URL: https://tu-servidor.com/ghl/call
# Headers: Content-Type: application/json
# Body (usar variables de GHL):

{
  "contact_phone": "{{contact.phone}}",
  "contact_name": "{{contact.name}}",
  "contact_id": "{{contact.id}}"
}

# Respuesta exitosa:
{ "success": true, "call_id": 1234567890 }
